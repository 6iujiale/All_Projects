    云端技术框架

    引言

    当我们使用Cruzr平台进行开发时，我们一定想知道Cruzr云端开放平台技术框架是怎么样的？Cruzr云端API有哪些？在使用云端API之前我们需要做哪些准备呢？这里简单介绍一下Cruzr云端平台的技术框架，让大家对Cruzr云端架构有一个简单的了解，弄清云端API的调用流程，以便大家快速掌握Cruzr云端API的接入。

    Cruzr云端开放平台框架图

    Cruzr云端开放平台的框架如下：

    Cruzr云端开放平台框架图
    由上图可知，Cruzr云端开放平台主要分为五层，从下到下分别是网关层、鉴权层、转发层、能力层和基础组件。网关层主要是负责所有的网络请求，反向代理和负载均衡等；

    鉴权层对所有云端API的调用进行鉴权，频次管控，及防篡改安全验证等等；

    转发层则对所有被允许调用的请求进行透传转发；

    能力层提供了丰富的云端API能力，包括基础信息能力、诊断信息能力、人脸识别能力、语音能力、运维统计能力、运动分析能力等；

    基础组件为所有的能力提供基础的组件服务，比如文件存储、日志管理、配置管理、监控预警、消息队列等等。

    如何使用

    步骤一：获取服务接入凭证

    第三方合作伙伴请联系销售申请接入凭证，进行资质审核，审核通过后，由平台方分配服务接入凭证、接口权限，并将相关审核结果通过邮件等方式发送给开发者。

    步骤二：服务开发和验证

    第三方开发者获取接入凭证后，参照云端API接入标准规范文档，结合自身的服务接口进行开发，并对开发完成的功能进行自测和验证。
